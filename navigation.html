Assumptions

  1. The breakpoints in this exercise are divided into two categories.

    Viewport breakpoints apply to the width of the browser window. Multiple viewport breakpoints mean the UI will adjust as the user resizes the browser.

    Device breakpoints apply to the width of the physical display. Device widths will change as a user rotates from landscape to portrait but are otherwise constrained.
  2. For this exercise we are considering the following breakpoints.

    • small devices is a device breakpoint restricted to <= 30em (480px) screen width.
    • medium viewport is a viewport breakpoint applied to devices with >= 59em (944px) viewport width.
    • large viewport is a viewport breakpoint applied to devices with >= 64em (1024px) viewport width.
    • x-large viewport is a viewport breakpoint applied to devices with >= 64em (1600px) viewport width.
  3. Screen and viewport widths will be defined using ems. For this exercise an em is roughly equal to 16px. Use pxtoem.com if you need convert measurements.

    Why ems?
    We use ems based on a historical issue where pixel based breakpoints would break when the user zoomed in and out of the browser.
